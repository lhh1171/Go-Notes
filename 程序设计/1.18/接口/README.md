# 接口

接口(`interface`) 是多个方法声明集合，代表一种调用契约。

&nbsp;

在设计上，接口解除显式类型依赖(DIP，依赖倒置)，提供面向对象多态性。</br>
定义小型、灵活及组合性接口（ISP，接口隔离），减少可视方法，屏蔽内部结构和实现细节。</br>

&nbsp;

只要目标类型方法集包含接口全部方法，就视为实现该接口，无需显示声明。</br>
当然，单个目标类型可实现多个接口。

&nbsp;

* 接口限制
    * 不能有字段。
    * 只能生声明方法，不能实现。
    * 可嵌入其他接口。

* 接口实现
    * 通常以 `er` 作为名称后缀。
    * 空接口(`interface{}`，`any`) 没有任何方法声明。
